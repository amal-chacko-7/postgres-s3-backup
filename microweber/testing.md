# Manual testing basics

### What is software testing?
Software testing is the process of checking whether an application works as expected and identifying defects before release.

### Why is testing important?
Testing ensures product quality, reduces defects, improves user satisfaction, and prevents business losses caused by software issues.

### What is manual testing?
Manual testing is testing software without automation tools, by executing test cases manually and comparing actual results with expected results.

### What are the advantages of manual testing?
It is easy to start, flexible, useful for exploratory and usability testing, and does not require scripting knowledge.

### What are the disadvantages of manual testing?
It is time-consuming, repetitive for large projects, and can be affected by human error.

### What is the difference between verification and validation?
Verification checks whether we are building the product correctly, while validation checks whether we are building the correct product.

### What is the difference between QA and testing?
QA is process-focused and ensures quality in the overall development process. Testing is product-focused and checks the actual application for defects.

# SDLC and STLC

### What is SDLC?
SDLC stands for Software Development Life Cycle. It is the process used to plan, design, develop, test, deploy, and maintain software.

### What is STLC?
STLC stands for Software Testing Life Cycle. It is the testing process that includes requirement analysis, test planning, test case design, environment setup, execution, defect reporting, and closure.

### What is the role of a tester in STLC?
A tester analyzes requirements, prepares test cases, executes tests, logs defects, verifies fixes, and supports test closure.

### At what stage should testing start?
Testing should start as early as possible, ideally from requirement analysis, to catch defects early.

# Test case and defect questions

### What is a test case?
A test case is a set of steps, input data, expected results, and conditions used to verify a specific requirement or functionality.

### What are the components of a good test case?
Test case ID, title, preconditions, test steps, test data, expected result, actual result, and status.

### What is a test scenario?
A test scenario is a high-level idea of what to test. A test case gives the detailed steps.

### What is a defect or bug?
A defect is a mismatch between expected and actual behavior in software.

### What information should a bug report contain?
Bug title, description, steps to reproduce, expected result, actual result, severity, priority, environment, screenshots, and status.

### What is the difference between severity and priority?
Severity shows how serious the defect is. Priority shows how soon it should be fixed.

### What is the bug life cycle?
New, assigned, open, fixed, retest, verified, closed. Sometimes it can also move to rejected, duplicate, deferred, or reopened.

### What is the difference between retesting and regression testing?
Retesting checks whether a specific bug is fixed. Regression testing checks whether the fix affected other existing features.

# Regression testing

### What is regression testing?
Regression testing is done to ensure that new changes or bug fixes have not broken existing functionality.

### When do we perform regression testing?

### Why is regression testing important?

# Common beginner testing questions

### What is a smoke test?
A quick test to verify that the main application functions are working after a build is deployed.

### What is sanity testing?
A focused test to check whether a specific fix or feature is working properly.

### What is exploratory testing?
Testing without predefined test cases, where the tester explores the application and checks behavior dynamically.

### What is positive testing?
Testing with valid input and expected user actions.

### What is negative testing?
Testing with invalid input or unexpected conditions to ensure the application handles errors correctly.

### What is boundary value analysis?
A test design technique where values at the edges of input ranges are tested, since defects often occur there.

### What is equivalence partitioning?
A: A technique that divides input data into valid and invalid groups so fewer test cases can cover more input conditions.
